# Documentación

La **especificación oficial** del proyecto vive en `OfficeSpace_Specification/docs/` (documentos 00 a 09), siendo `09_Decisiones_Aprobadas.md` el documento canónico que prevalece en caso de conflicto.

Esta carpeta (`officespace/docs/`) se reserva para documentación técnica generada durante la implementación (p. ej. `ARCHITECTURE.md`, `API_CONTRACT.md`, guías de instalación/despliegue/pruebas), conforme avancen las fases.

Documentos de auditoría y planificación (en la raíz del proyecto):
- `Reporte_Auditoria_Documentacion.md`
- `Reporte_Auditoria_Verificacion.md`
- `Fase2_Plan_Implementacion.md`
